# 吃豆人游戏 - 完整设计文档

## 项目概述

基于 **p5.js** 实现的经典吃豆人游戏，完整实现幽灵 AI 系统和 A* 路径规划算法。

- **技术栈**：p5.js (创意编程库) + JavaScript
- **项目属性**：独立完成

---

## 一、幽灵 AI 行为系统

### 1. 红色幽灵：激进追击型

- **策略**：始终追踪吃豆人的当前位置
- **特点**：
  - 速度随游戏进度线性增加（吃的豆子越多，速度越快）
  - 最直接的追击者，给予玩家持续压力
  - 在特殊能量豆效果期间变为恐惧状态

### 2. 粉色幽灵：预判拦截型

- **策略**：追踪吃豆人前方 4 个单位的预判位置
- **逻辑**：如果吃豆人向右移动，粉色幽灵会瞄准吃豆人右边 4 格的位置
- **目的**：在吃豆人前进方向上设置拦截，防止玩家直线逃跑

### 3. 青色幽灵：协同包围型

- **策略**：结合红色幽灵位置进行计算
- **计算方法**：计算吃豆人到红色幽灵的向量，将这个向量从吃豆人位置延伸出去，瞄准延伸后的位置
- **效果**：与红色幽灵形成包抄态势

### 4. 橙色幽灵：巡逻警戒型

- **策略**：根据与吃豆人的距离切换行为模式
- **距离逻辑**：
  - 距离 > 8 格：追击模式
  - 距离 ≤ 8 格：巡逻模式
- **目的**：防止玩家躲在角落，保持地图控制

## 二、状态管理系统

### 1. 三种主要状态

| 状态 | 说明 |
|------|------|
| 追击状态（Chase） | 主动追踪吃豆人 |
| 分散状态（Scatter） | 前往各自的地图角落 |
| 恐惧状态（Frightened） | 吃豆人吃特殊能量豆后的逃跑状态 |

### 2. 状态切换机制

- 吃豆人吃特殊能量豆 → 所有幽灵进入恐惧状态
- 恐惧时间结束 → 返回之前的状态
- 随机切换（小概率）：追击 ↔ 分散

### 3. 恐惧状态特点

- 移动速度降低 30%
- 颜色变为蓝色并且闪烁
- 被吃后会在起点重生

## 三、能量豆系统

### 1. 普通能量豆

- **效果**：增加分数
- **胜利条件**：当所有豆子被吃完，游戏胜利

### 2. 特殊能量豆

- **位置**：位于迷宫的特殊数据加密节点
- **效果**：激活后使吃豆人获得临时高级权限，可重写幽灵协议
- **表现形式**：幽灵变为蓝色闪烁状态，吃豆人可以吃幽灵

## 四、核心算法：A* 寻路算法

### 算法思想

A* 算法是一种启发式搜索算法，旨在找到从起点到目标点的最短路径。它通过综合考虑已经走过的距离和预估剩余距离来选择最优路径。

### 三个关键概念

| 概念 | 含义 |
|------|------|
| G 值（实际成本） | 从起点到当前节点的实际移动成本 |
| H 值（启发式成本） | 从当前节点到目标点的预估成本 |
| F 值（总成本） | F = G + H，用于评估节点的优先级 |

### 启发式函数

使用**曼哈顿距离**作为启发式函数：

> h(n) = |x1 - x2| + |y1 - y2|

### 搜索过程

1. 维护两个列表：开放列表（待检查节点）和关闭列表（已检查节点）
2. 从起点开始，逐步探索相邻节点
3. 每次选择 F 值最小的节点进行扩展
4. 如果找到目标节点，通过回溯父节点得到完整路径
5. 如果开放列表为空仍未找到目标，说明无路径可达

### 性能优化

- 限制寻路范围（只计算一定步数内的路径）
- 缓存路径，减少重复计算
- 利用 p5.js 的 `draw()` 循环实现实时路径更新

## 五、p5.js 在本项目中的应用

| 功能 | p5.js 函数 | 用途 |
|------|------------|------|
| 游戏循环 | `draw()` | 每帧更新游戏状态和渲染画面 |
| 键盘控制 | `keyPressed()` | 监听玩家方向键输入 |
| 图像绘制 | `image()`、`rect()`、`circle()` | 绘制迷宫、吃豆人、幽灵、豆子 |
| 碰撞检测 | 基于网格坐标计算 | 判断吃豆人与墙壁、豆子、幽灵的碰撞 |
| 定时器 | `setTimeout()`、`millis()` | 控制恐惧状态持续时间 |
